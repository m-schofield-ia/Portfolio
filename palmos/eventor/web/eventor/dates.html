How to enter dates quickly in Eventor.

When you create a new event in Eventor you need to set, among other things, the date of the event. Eventor uses the standard Date Selector to do this. The Date Selector is in normal operations pretty good - but if you need to add a lot of events starting before, say, year 2000, you have to tap a lot of times on the little left arrow in the Date Selector, as the Date Selector have no provisions for manual date entry.

This, as a lot of people have found out, is quite annoying. It will be addressed in the next release of Eventor.

There is, however, a quick work around:

  1. Install the Eventor Conduit (download it from the Eventor Mainpage).
  2. Enter the events in Eventor. Set the date in the Date Selector to "Today".
  3. Perform a HotSync. You will now have a Eventor data directory in your Palm HotSync directory containing two files, EventorData.BAK and EventorData.dat. Your Palm HotSync directory is normally C:\Palm\{your hotsync name}\ or C:\Program Files\Palm\{your hotsync name}.
  4. Open the EventorData.dat file in notepad and make your changes as described below.
  5. Perform yet another HotSync - this will transfer your changes to your handheld.
  6. Voila! You're done.

How to change the EventorData.dat file

The EventorData.dat file is made up of two sections. The first section lists all the categories and meta data about the categories. The second section lists all the events and meta data about the events. The following (short) file will be used as an example: 

ev3
2
1       1       0       Happenings
2       2       0       Birthdays
3
13541377        2       -       1973-3-17       YY      7       Charlotte
13541378        2       -       1997-12-9       YY      7       Kristian
13541379        1       -       2009-12-24      YY      7       Christmas Eve, in 6 years

The first line, ev3, is a fixed identification string. The second line, 2, is the number of categories. Line 3 and 4 is the data describing the categories. Line 5, 3, is the number of events. Starting on line 6 follows the events.

Each event line is made up of 7 colums separated by TABs (tabulations): 

ID      Category   Attributes    Date    Flags    Days    Event

Columns 3 (Attributes) and column 4 (Date) are of interest to us. The Date column lists the date of the event in YYYY-{m}M-{d}D format. The year value is always four digits. The month and day values may be one or two digits.

To change the date of an event change the date in column 4. Then put a uppercase M in column 3 to indicate to the HotSync process that the record/line has been modified.

Do not, however, change any other values.

Example. Suppose that I need to change the date for Charlottes birthday to 1973-3-18. I would change line 6 in the file to read: 

13541377        2       M       1973-3-18       YY      7       Charlotte
