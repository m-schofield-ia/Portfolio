0L Systems User Manual


Introduction

0L Systems is a simple implementation of a 0L System renderer. A 0L System, or more formally a L System, is a string rewriting system which produce a long string to be interpreted as commands for a LOGO turtle.



Installation

Installation of 0L Systems is no more different than installation of any other Palm OS program:

  1. Download [1] 0L Systems.
  2. Unpack the p0ls.zip archive using WinZip [2] or UnZip [3].
  3. Start the standard Palm Install tool. Select 'Add', then browse to where you unpacked the p0ls.zip archive. Highlight the p0ls.prc file and select 'Open'. Then select 'Done'.
  4. Start the HotSync Manager if it is not running.
  5. Insert your Palm PDA in the craddle, then press the HotSync button on the craddle.

When the synchronization is done you will have a '0L Systems' icon in the Unfiled category on your Palm PDA. Feel free to move 0L Systems to another category.



Theory

Welcome to the boring part - I will keep it short :)   As said in the introduction a L System is a string rewriting system which, based on a user-defined Axiom and possible a set of Production Rules, generates a long commend string suitable for interpretation by a LOGO-like turtle.

Using 0L Systems or more advanced L Systems it is easy to create f.ex. advanced trees and bushes given a short Axiom and a few production rules.

So how does it work? Let me show you by using the classic Von Koch Snowflake curve as an example. The Von Koch Snowflake curve is shown on the frontpage in its sixth stage.

Imagine a simple LOGO turtle capable of these commands:

  • F    draw a line forward.
  • +    turn right by 60°
  • -    turn left by 60°

We start out with a straight line. This is written as F. In all stages we substitute all straight lines with the sequence:

forward, turn left, forward, turn right twice, forward, turn left, forward

This, the Production Rule, is written as:

F-F++F-F

Formally:

F  ->  F-F++F-F

After stage one our Axiom (F) becomes F-F++F-F - see picture 1 below.

The output from stage 1 is used as Axiom in stage 2. Again we substitute with the sequence above getting:

F-F++F-F - F-F++F-F ++ F-F++F-F - F-F++F-F

This is shown in picture 2 below.

The output from stage 2 is used as Axiom in stage 3. After substitution:

F-F++F-F - F-F++F-F ++ F-F++F-F - F-F++F-F -
F-F++F-F - F-F++F-F ++ F-F++F-F - F-F++F-F ++
F-F++F-F - F-F++F-F ++ F-F++F-F - F-F++F-F -
F-F++F-F - F-F++F-F ++ F-F++F-F - F-F++F-F

... and so on. As you can see from the pictures below, the pictures are getting richer and richer on details.

Turtle commands

The turtle understands a basic set of commands:

  • F    draw a line forward.
  • f    move a step forward.
  • +    turn right by the number of degrees given on the Axiom form.
  • -    turn left by the number of degrees given on the Axiom form.
  • |    turn 180° (turn back).
  • [    save turtle state on stack (coordinates and direction).
  • ]    restore turtle state from stack (coordinates and direction).

Working with 0L Systems

When starting 0L Systems you are met with the Axiom form:

  • Depth: this is the number of iterations to be done. Depending on the version of the OS, amount of RAM and complexity of Axiom and Production Rules this can be as low as 3 or 4 and as high as 13 or 14 ... or even higher!
  • Angle: this is the angle to turn by a single + or - command.
  • Axiom this is where you enter the Axiom (* surprise, surprise *).

Press Rules to go to the Production Rules form. Press Load to load one of the default L Systems installed.

On the Production Rules form you enter the various production rules:

You can override the meaning of the F and f commands - this is f.ex. done in the Von Koch Snowflake curve.

Press Generate to start rendering. Press Axiom to go back to the Axiom form - input written on the Axiom and Production Rules forms is saved between the forms. So you can switch back and forth between these forms without loosing input. Press Clear to clear all fields on the Production Rules form.

Press Cancel to abort.

When the string has been rendered, you have the option to save the data used to create the string.



Hints

If you during string expansion gets the dreaded "Out of memory" box try to lower the depth.

Using the default 0L Systems as base for your own experiments seems like a good idea.

Strange angles usually results in strange drawings unless used in conjunction with the push and pop commands.
