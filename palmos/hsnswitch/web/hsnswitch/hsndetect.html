HSNDetect v1r1

If HSNSwitch fails to run it may be that it cannot parse the HotSync record. As the HotSync record is undocumented a bit of reverse engineering must be done by using HSNDetect.

HSNDetect is a tool which collects the relevant system information and stores them in a database. The database should be mailed to me - instructions are in the README.txt in the archive.

When I have the collected information I might get a better understanding on how the HotSync record are stored on your hardware platform.

  • Download HSNDetect - read the README file before installing!
  • Download the source code archive.


Manual

To use HSNDetect follow these steps:

  1. Install HSNDetect
  2. Launch it - it may take a while to complete. On my Tungsten T it runs for almost 2 minutes. The slot count goes up to 65535.
  3. When the program has stopped do a full HotSync.
  4. Email me the resulting HSND_HSNDetectData.pdb file (usually found in c:\Program Files\palmOne\{id}\backup).
  5. Also include information about your Hardware platform (like Tungsten T3, Treo 650, ...)

I will then have a better chance of getting HSNSwitch working on your hardware platform.
